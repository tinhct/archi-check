# Security Policy

## Reporting a Vulnerability

We take security issues seriously. If you find a security vulnerability, please do NOT report it in public GitHub issues. Instead, report it privately via GitHub Security Advisories.

## Disclosure Policy (90-Day Embargo)

To protect deployments running ArchiCheck, we follow a strict **90-day embargo policy**:
1. Once a vulnerability report is received, we will work with you to analyze and confirm the issue.
2. We will draft a patch and verify the fix.
3. The vulnerability details will remain under embargo (private) for 90 days from confirmation or until a patch is widely deployed, whichever comes first, giving self-hosted enterprise installations sufficient time to update.
4. After the embargo period, the security advisory will be published publicly.

## Supported Versions

Security patches are actively applied to:
- The current major version (`v1.x.x`)
- The immediate prior major version (`v0.x.x`)
